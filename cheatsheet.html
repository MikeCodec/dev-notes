🖤 Git Workflow – Paso a Paso

1️⃣ Configuración Inicial

Define tu nombre y correo para que GitHub muestre correctamente quién hace cada commit.

git config --global user.name "Mikecodec" git config --global user.email "michaelfloresrojas31@gmail.com"

2️⃣ Inicializar Repositorio

git init # Crea un repositorio local # Opcional: crear archivo .gitignore y agregar archivos a ignorar

3️⃣ Primer Commit y Conexión con GitHub

git add . # Agrega todos los archivos git add archivoEspecifico.js # O agregar un archivo específico git commit -m "Primer commit" # Confirmar cambios git remote add origin https://github.com/MikeCodec/repositorioPrueba.git # Conecta con GitHub git push -u origin master # Subir proyecto por primera vez

4️⃣ Trabajar con Ramas

git checkout -b nuevaRama # Crear (local) y cambiar a nueva rama git add . o git add archivoEspecifico.js # Agregar cambios git commit -m "Descripcion de los cambios" # Confirmar cambios git push --set-upstream origin nuevaRama # Crear rama en GitHub y subir git push -u origin nuevaRama # Alternativa para subir rama git checkout nombre-Rama # Cambiar a otra rama

5️⃣ Clonar Repositorio GitHub

git clone URL-del-repositorio # Clona el repositorio git branch -r # Ver todas las ramas locales y remotas git checkout -b nombreRama origin/nombreRama # Trabajar con una rama específica

6️⃣ Fusionar Cambios

git checkout nombreRamaDestino # Selecciona rama donde aplicar cambios git merge nombreRamaOrigen # Fusiona cambios git push origin nombreRamaDestino # Subir cambios a GitHub

7️⃣ Actualizar Archivos en GitHub

git add . o git add archivoEspecifico.js # Agregar cambios git commit -m "Descripcion de los cambios" # Confirmar cambios git push # Subir actualización a GitHub # Opcional: eliminar .git de PC Remove-Item -Recurse -Force .git git status
💡 Tip: Evita usar comandos de borrado de todo el repositorio a menos que sea necesario. Siempre revisa el estado con git status antes de push.
