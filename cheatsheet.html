🖤 Git Workflow – Paso a Paso

1️⃣ Configuración e Inicialización

UsuarioEmail: Define tu nombre y correo y crea el primer commit para que GitHub registre correctamente tus cambios.
git config --global user.name "Mikecodec" git config --global user.email "michaelfloresrojas31@gmail.com" git init # Crea un repositorio local # Opcional: crear archivo .gitignore y agregar archivos a ignorar git add . # Agrega todos los archivos git add archivoEspecifico.js # O agregar un archivo específico git commit -m "Primer commit" # Confirmar cambios git remote add origin https://github.com/MikeCodec/repositorioPrueba.git # Conecta con GitHub git push -u origin master # Subir proyecto por primera vez

2️⃣ Trabajar con Ramas

Ramas: Crea y trabaja con ramas locales y remotas para organizar tu desarrollo y mantener el historial limpio.
git checkout -b nuevaRama # Crear (local) y cambiar a nueva rama git add . o git add archivoEspecifico.js # Agregar cambios git commit -m "Descripcion de los cambios" # Confirmar cambios git push --set-upstream origin nuevaRama # Crear rama en GitHub y subir git push -u origin nuevaRama # Alternativa para subir rama git checkout nombre-Rama # Cambiar a otra rama

3️⃣ Clonar Repositorio GitHub

Clonado: Trae un repositorio remoto a tu máquina y trabaja en ramas específicas.
git clone URL-del-repositorio # Clona el repositorio git branch -r # Ver todas las ramas locales y remotas git checkout -b nombreRama origin/nombreRama # Trabajar con una rama específica

4️⃣ Fusionar Cambios

Fusión: Aplica cambios de una rama a otra para mantener tu proyecto actualizado y consolidado.
git checkout nombreRamaDestino # Selecciona rama donde aplicar cambios git merge nombreRamaOrigen # Fusiona cambios git push origin nombreRamaDestino # Subir cambios a GitHub

5️⃣ Actualizar Archivos en GitHub

Actualización: Sube cambios confirmados al repositorio remoto y verifica el estado de tu proyecto.
git add . o git add archivoEspecifico.js # Agregar cambios git commit -m "Descripcion de los cambios" # Confirmar cambios git push # Subir actualización a GitHub Remove-Item -Recurse -Force .git git status
💡 Tip: Evita usar comandos de borrado de todo el repositorio a menos que sea necesario. Siempre revisa el estado con git status antes de push.
